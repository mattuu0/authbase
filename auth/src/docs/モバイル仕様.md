# モバイルアプリ連携用 API 利用仕様書

## 概要
モバイルアプリへの認証トークン受け渡しにおいて、直接アクセストークンをURLに含めるのではなく、安全な「ブリッジトークン（JWT）」方式を採用します。

## 認証フロー

### 1. ブリッジトークンの発行
ログインページ（Web）でログイン成功後、アプリへのリダイレクト時に使用する一時トークンを取得します。

- **Endpoint**: `POST /bridge/issue`
- **Auth**: セッション認証（リフレッシュトークン/セッションが必要）
- **Response**:
  ```json
  {
    "bridge_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
  ```
- **利用方法**: 取得した `bridge_token` を URL スキームに含めてモバイルアプリを起動します。
  - 例: `authbase://?bridgetoken=<bridge_token>`

---

### 2. トークン交換（アクセストークンの取得）
モバイルアプリ起動後、アプリ側でブリッジトークンを本番用アクセストークンに交換します。

- **Endpoint**: `GET /bridge/exchange`
- **Header**:
  - `Authorization`: `Bearer <bridge_token>`
- **Response**:
  ```json
  {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
  ```

## セキュリティに関する重要事項

1. **トークンの検証**: `/bridge/exchange` エンドポイントでは、ブリッジトークン（JWT）の署名、期限（5分）、およびDB上での使用済みフラグ（使い捨て）を厳密に検証します。
2. **使い捨て保証**: ブリッジトークンは、一度交換に使用されると即座に無効化されます。同一トークンでの二重交換は拒否されます。
3. **有効期限**: ブリッジトークンの有効期限は発行から5分間です。
